Identifying Usability Concerns

Usability can be termed by tweaking such as learnability, efficiency factors, I have did in a good way to make design more optimized. To spread out more legibly first of all will understand on why we use Boolean methods to differentiates things. I found AND, OR, NOT and it is majorly used in advanced search of Recruit CRM to search the employees widely. There is a constraint in advanced Boolean search where OR, NOT are used in block letters.

In the above image, they claim to the new way of Boolean by removing brackets and quotes but it only works when it is in block letters. If the user needs some employee which is very diverse in technical specification then it is very helpful to write prompt with OR, NOT.

I did this test as a free trail with 50 candidate profile.

Canada OR TurkeyNo contains search Result found
canada or turkeycontains search No result found
canada OR turkeyNo contains search Result found

Boolean expressions with case sensitive can make a chaos for users and I personally felt it at very first time when I tried. This may miss some occurrences while typing a long Boolean string for all columns.

Ideation - Optimizing search

After uploading all the candidate profiles, the system should go through all profile and update the necessary details with search filters. As for example the company is in China and has no other office spaces in anywhere then, those cities, states should be updated in system. Basically I am combining the Boolean search and the filter search methods into one, So it will more easy to find the employees by tapping down instead of framing them in SQL methods.
